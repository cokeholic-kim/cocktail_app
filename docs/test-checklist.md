# 핵심 경로 수동 QA 체크리스트 (P1)

본 문서는 배포/PR 전 핵심 사용자 경로의 재현성 높은 수동 점검 기준을 정리한다.

## 1. 실행 조건
- 브라우저: Chrome 또는 Edge 최신 1개 이상
- 환경: `npm install` 후 `npm run dev` 또는 스테이징 URL
- 사전 조건:
  - 빌드 통과: `npm run build`
  - `BASE_URL` 원본 값(또는 환경별 기본값) 기록
  - 브라우저 콘솔/네트워크 탭에서 로그 수집 가능

## 2. 핵심 경로 점검
대상 경로
- `/`
- `/cocktails`
- `/cocktails/[name]`
- `/ingredients`
- `/ingredients/[name]`
- `/myingredients`
- `/login`
- `/join`

확인 항목(각 경로 1회 이상)
- HTTP 200 또는 의도된 리다이렉트 확인
- 레이아웃/내비게이션/버튼 등 UI 깨짐 없음
- 로딩/빈 응답/에러 메시지 표출 상태가 비정상 종료 없이 동작
- 라우팅 전환과 주요 입력 동작 1회 이상 수행

## 3. 실패 케이스 점검
아래 케이스는 모두 실패 유발 후 복구(원복)까지 포함해서 기록한다.

- 백엔드 미연결(오프라인)
  - `BASE_URL`을 임시 미해결 도메인으로 변경
  - 대상: `/`, `/cocktails`, `/ingredients`, `/myingredients`
  - 확인: 화면이 크래시 없이 fallback/안내 메시지 유지
  - 원복:
    - `BASE_URL`을 원본 값으로 복원
    - 네트워크 차단 설정 해제
    - 서버 모킹/프록시/개발 프록시 설정이 있으면 원복 또는 중단

- 백엔드 빈 응답
  - API 응답을 빈 바디 또는 `body: []`로 강제
  - 확인: 빈 응답에서 빈 목록/안내 메시지로 정상 fallback
  - 원복:
    - 모킹/프록시 응답을 정상 응답으로 복원
    - `BASE_URL` 및 테스트 헤더 설정 원복

- API 오류(4xx/5xx) 강제
  - 해당 API 요청을 실패 응답으로 강제
  - 확인: 에러 상태에서 화면이 죽지 않고 재시도/재진입 가능
  - 원복:
    - `BASE_URL` 원본 복원
    - 실패 유도한 네트워크 규칙 및 모킹/스크립트 중단

- 라우팅 인코딩 오류
  - 영문/공백/특수문자 재료/칵테일 이름으로 진입
  - 확인: URL 인코딩/디코딩 후 페이지 렌더링 정상
  - 원복:
    - 테스트 입력 값을 정상 값으로 되돌림
    - 캐시/로컬 상태를 초기화하고 페이지 새로고침

## 4. 스크린샷·로그 수집 규칙
- 항목별 최소 1회 이상 스크린샷(성공/실패) 수집
- 실패 재현 시 다음 네트워크 증적을 함께 수집:
  - 요청 URL
  - 응답 상태 코드
  - 응답 바디(또는 에러 코드/에러 응답 메시지)
  - (가능하면) 요청 헤더 주요 항목
- 수집한 로그/네트워크 증적은 QA 댓글 또는 PR 본문에 첨부

## 5. 통과 기준
- 핵심 경로 1회 통과 + 실패 케이스 4개 항목 통과
- 네트워크 장애 후 재시도 버튼 또는 재호출로 크래시 없이 화면 복구 가능
- 앱 새로고침 없이 임시 실패 상태에서 기본 화면으로 복귀 가능(치명 오류 오버레이 미발생)
- 동일 세션 내에서 연속 시나리오 실행 시 이전 시나리오 설정 잔영향 없이 종료
- PR 본문 `수동 QA` 항목에 링크·로그 위치·원복 결과 포함
