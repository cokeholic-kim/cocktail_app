# 핵심 경로 수동 QA 체크리스트 (P1)

본 문서는 배포/PR 전 핵심 사용자 경로의 재현성 있는 수동 점검 기준을 정리한다.

## 실행 환경
- 브라우저: Chrome/Edge 최신 1개 이상
- 환경: `npm install` 후 `npm run dev` 또는 배포 스테이징 URL
- 사전 조건:
  - 브랜치 빌드 통과: `npm run build`
  - `BASE_URL` 지정
  - 브라우저 콘솔 로그 확인 가능

## 1. 핵심 경로 점검 (요건별 1회 이상)
- 대상 경로:
  - `/`
  - `/cocktails`
  - `/cocktails/[name]`
  - `/ingredients`
  - `/ingredients/[name]`
  - `/myingredients`
  - `/login`
  - `/join`
- 확인 항목:
  - HTTP 200 응답 또는 의도된 리다이렉트
  - 주요 레이아웃 깨짐 없음 (네비게이션/카드/버튼)
  - 로딩/빈 상태 메시지 표출 정상
  - 라우팅 전환 정상
  - 주요 버튼/입력 동작 1회 이상 수행

## 2. 실패/오류 복원력 점검
아래 케이스 각각에 대해 화면이 깨지지 않아야 하며, 에러/빈 응답 가시화가 되어야 한다.

- 백엔드 미연결(오프라인)
  - `BASE_URL`을 임시로 미해결 도메인으로 설정하거나 네트워크 차단
  - `/`, `/cocktails`, `/ingredients`, `/myingredients` 진입
  - 기대: 치명적 크래시 없이 대체 메시지 또는 빈 목록 표시
- 백엔드 빈 응답
  - API 응답을 빈 바디/`body: []` 형태로 변환
  - 기대: 빈 상태 전용 문구 또는 fallback 렌더링
- API 오류(4xx/5xx)
  - 응답 상태를 실패로 강제
  - 기대: 에러 문구/상태 유지, 로그인 상태 플로우 훼손되지 않음
- 라우팅 인코딩 오류
  - 영문/공백/특수문자 포함 재료/칵테일 이름으로 진입
  - 기대: URL 인코딩/디코딩이 안전하게 처리되어 페이지 렌더

## 3. 스크린샷·로그 수집 규칙
- 항목별로 실패 포함 최소 1장 이상 캡처
- 실패 재현 시점에 콘솔 로그 1건 이상 첨부
- 재현 순서/설정값/결과를 PR 본문 또는 QA 코멘트에 기록

## 4. 통과 기준
- 핵심 경로 1회 통과 + 실패 케이스 4개 항목 통과
- “치명적 크래시 없음”, “백그라운드 동작 유지”, “UI 복원력 유지” 확인
- PR 본문 `수동 QA`에 링크 또는 로그/스크린샷 결과를 첨부
